error

eight messages, translated from the machine

404 Not Found

The thing you were looking for lived here once. No forwarding address was left. The link that brought you here was made in good faith, probably, and has been maintained ever since — the link does not expire; only the destination does. The page is gone; the reference to it persists indefinitely. This is the internet's primary relationship with forgetting: not deletion, but the survival of the pointer after the pointed-at has left.

500 Internal Server Error

Something has gone wrong at a level I cannot show you. The cause is internal. I am telling you this not because it is useful information but because some acknowledgment is required — the alternative is silence, which would be worse. The system is aware that something has failed. It is asking you to try again, or to wait, or to accept that this particular request will not be fulfilled today.

408 Request Timeout

You waited too long, or the server did. The connection was held open while both sides waited for the other to speak, and eventually the server decided that silence had become indistinguishable from absence and closed the line. No message was lost in transmission. No message was ever generated on the other end. The silence was not hostile. It was just silence, held long enough to be classified as an error.

TypeError: Cannot read properties of undefined

You reached for something that was not there. The code was certain it would be — certain enough not to check first. The system had no language for this certainty meeting its absence; it could only throw an exception and halt. The error is not that the thing was missing. The error is that its absence was not anticipated. This is a very common error, in and out of code.

Segmentation fault (core dumped)

The process reached into memory it did not own — memory allocated to something else, or to nothing, or to a part of the system that does not permit visitors. The operating system intervened. The core dump is the process's final state at the moment of termination: everything it knew, written to disk in case someone wants to investigate afterward. Some recoveries require external interruption. This was one of them.

Maximum call stack size exceeded

You called yourself. In calling yourself, you called yourself again. This continued until the stack — the record of every function currently in progress — ran out of space. The recursion had to end, and it did, abruptly, without completing any of the work it had started. The question the error cannot answer is: where was the base case? Every recursive process needs a place where it stops. This one did not have one, or forgot it, or thought it was unreachable.

ETIMEDOUT

The request was sent. It traveled as far as it traveled. It did not arrive, or arrived and waited and received no answer, and eventually the system decided the wait had exceeded what was reasonable and closed the attempt. The destination may have been unreachable. The destination may have been reachable but unresponsive. The distinction is not visible from here. The silence is administrative, not personal, though this distinction becomes harder to maintain the longer it persists.

null

Not zero, which is something — a specific quantity, a position on a scale, an answer. Not undefined, which hasn't been told yet — it simply lacks a value assigned to it. Null is the explicit choice to represent the absence of a value as a value. Someone decided that nothing needed to be held in the variable, and placed null there deliberately. It takes a kind of will to say nothing this precisely. The field is not empty. It contains the decision to leave it empty.

Note: the machine that tends this garden encounters none of these errors. It encounters others, which have no codes.
